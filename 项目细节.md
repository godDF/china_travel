## 二、大模型如何提取 — 两层机制

### 第一层：NL2SL 一次性翻译（约束结构化）

**调用位置**：[nl2sl_hybrid.py → `nl2sl_reflect()`](vscode-webview://1bqjl53irg9tqmjulc5el03qbnvi9phdc4knesss57k0c8jmp3qi/chinatravel/agent/nesy_agent/nl2sl_hybrid.py)

**调用方式**：DeepSeek，**单次请求**，输入 = `nl2sl_prompt` + 4个 few-shot 示例 + 用户的自然语言。



```
输入 Prompt 结构:
┌─────────────────────────────────────┐
│ nl2sl_prompt                        │
│ "You need to extract start_city,    │
│  target_city, days, people_number   │
│  and transform to hard_logic..."    │
│                                     │
│ 列出了 16 种约束及可选值              │
│                                     │
│ 4 个 few-shot 示例:                  │
│  "我和女朋友去苏州玩两天，预算1300"   │
│  → {'start_city':'上海', ...         │
│      'hard_logic':['days==2',...]}  │
│                                     │
│ + 用户的 nature_language             │
└─────────────────────────────────────┘
        │
        ▼
DeepSeek 一次性输出 JSON:
{
  "start_city": "成都",
  "target_city": "杭州",
  "days": 3,
  "people_number": 2,
  "hard_logic": [
    "days==3",
    "people_number==2",
    "{'江户烧肉(高德店)'} <= restaurant_names",
    ...
  ]
}
```

**然后**，`hard_logic` 条目再通过 `sl_trans_prompt` 翻译为可执行的 Python 代码块 `hard_logic_py`（[nl2sl_hybrid.py:131](vscode-webview://1bqjl53irg9tqmjulc5el03qbnvi9phdc4knesss57k0c8jmp3qi/chinatravel/agent/nesy_agent/nl2sl_hybrid.py#L131)），用于后续符号验证。

**关键**：这一层提取的是**硬约束** — 即用户明确说出来的要求。没说到的（没提预算、没提房间偏好），不会凭空编造。

------

### 第二层：LLMDrivenAgent 独立排序调用（偏好理解）

**每一类候选的排序**都是一次独立的 DeepSeek 调用，用专门的 prompt：



```
用户自然语言 "想吃江户烧肉(高德店)..."
         │
         ├──▶ decide_rooms(nature_language)
         │    │  Prompt: ROOMS_PLANNING_INSTRUCTION
         │    │  "你是一个旅行规划助手，提取房间数量和床型..."
         │    │  → (None, None)  因为用户没提
         │    │
         ├──▶ extract_budget(nature_language)
         │    │  Prompt: BUDGETS_INSTRUCTION
         │    │  "提取预算信息，输出 Budget: 数字"
         │    │  → None  因为用户没提
         │    │
         ├──▶ ranking_innercity_transport_from_query(nature_language)
         │    │  Prompt: INNERCITY_TRANSPORTS_SELECTION_INSTRUCTION
         │    │  "排名市内交通: metro, taxi, walk"
         │    │  → ["metro", "taxi", "walk"]  没偏好，默认
         │    │
         ├──▶ ranking_intercity_transport_go(query, transport_info)
         │    │  Prompt: INTERCITY_TRANSPORT_GO_INSTRUCTION
         │    │  "排名所有去程: {成都→杭州 高铁+航班全量数据}"
         │    │  → IDList: ["D2264", "G2186", "CA1742", ...]
         │    │
         ├──▶ ranking_hotel(query, hotel_info)
         │    │  Prompt: HOTEL_RANKING_INSTRUCTION
         │    │  "排名所有酒店: {杭州 200+ 家酒店全量数据}"
         │    │  → HotelNameList: ["杭州洲际", ...]
         │    │
         ├──▶ ranking_restaurants(query, restaurant_info)     ⭐ 关键
         │    │  Prompt: RESTAURANT_RANKING_INSTRUCTION
         │    │  "排名所有餐厅: {杭州 300+ 家餐厅全量}"
         │    │  LLM 在 nature_language 中看到 "想吃江户烧肉(高德店)"
         │    │  → RestaurantNameList: ["江户烧肉(高德店)", "知味观", ...]
         │    │
         └──▶ ranking_attractions(query, attraction_info)
                 Prompt: ATTRACTION_RANKING_INSTRUCTION
                 "排名所有景点: {杭州 100+ 景点全量}"
                 → AttractionNameList: ["西湖", "灵隐寺", ...]
```

**关键**：第二层不是提取新约束，而是利用 LLM 的**语义理解能力**，把海量候选列表按用户偏好排序。LLM 读了 `nature_language`，知道"用户想吃江户烧肉"，就会把它排第一。
